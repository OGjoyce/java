Especificaciones Generales

El proyecto JPEG Image Handler trata de que usted aplique los conceptos aprendidos en clase de listas encadenadas y arreglos (bidimensionales) en el manejo de imagenes. Usted estara trabajando con imagenes originales con formato JPEG, las cuales deben ser trasladadas a formato BMP, manipuladas en ese formato y convertidas de regrso a JPEG.

Parte I: JPEG a BMP y viceversa
La primera tarea que debe poder hacer su proyecto es poder leer una imagen en formato JPEG y convertirla a BMP, y viceversa. Para esta parte del proyecto (y solamente en las partes que se le indiquen), se le permite utilizar las clases ImageIO y BufferedImage ya definidas en Java.
Utilizando estas clases, usted debe

  • (1) Leer una imagen JPEG y generar una copia de la imagen en formato BMP.
  • (2) Leer una imagen BMP y generar una copia de la imagen en formato JPEG.
Las imagenes convertidas a otro formato deben llevar el nombre de la imagen original con el prefijo converted, de tal manera que si su imagen se llama colores.jpeg, la imagen generada en formato bmp se llamaria converted-colores.bmp.

Mas adelante se le indicara como se ejecutara esta funcionalidad en su programa.

Parte II: Copia - JPEG
Para esta parte del proyecto debe poder leer una imagen en formato JPEG y generar una copia de la misma. La copia no debe ser manipulada de ninguna manera. En este caso debe utilizar las clases de Java antes mencionadas para poder leer la imagen jpeg, sin embargo no las puede utilizar para realizar la copia. Para esto debe hacer lo siguiente:

  • (1) Lea la imagen JPEG utilizando las clases de Java
  • (2) Convierta la imagen a BMP
  • (3) Cree una copia de esa imagen (BMP), utilizando la clase que se le provee
  • (4) Convierta esa copia a JPEG
Dentro del material que se le provee en para este proyecto, hay una clase BMPHandlerCopy, la cual debe usar para hacer la copia.

La imagen copia debe llevar el nombre de la imagen original con el prefijo copia, de tal manera que si su imagen se llama colores.jpeg, la imagen copia se llamaria copia-colores.jpeg.

Parte III: Rojo - Verde - Azul - Sepia

En la primera parte del proyecto, usted entendera como estan formados los colores en cada pixel. Su objetivo es separar su imagen original en sus tres tonos, rojo, verde y azul.

Imagen:
Usted va a trabajar inicialmente con una imagen en formato JPEG, esta debe convertirla a BMP, manipularla de esa manera y luego generar apartir de imagenes BMP, las imagenes JPEG corresponientes. A continuacion se le explica como esta definida una imagen BMP.
Las imagenes estan divididas en pixels, en cada uno de los cuales se guarda un color especifico dependiendo de su posicion en la imagen. Este color, se forma por tonalidades de rojo, verde y azul. La tonalidad (cantidad de color) es un numero del 0 al 255. Por ejemplo si un pixel fuera a guardar el rojo puro, este se codificaria como R=255 - V=0 - A=0, que significa que el color se representa por 255 de rojo, 0 de verde y 0 de azul.

Estructura del Proyecto

Para este proyecto se le proveen las siguientes clases en MaterialPJ1.zip :

  • ImageHandler: Clase abstracta de la cual todos los handlers de este proyecto deben heredar, y ser forzados a implementar los metodos de lectura de imagen original y generacion de imagenes.
  • BmpHandlerCopy: Handler que hereda de ImageHandler y crea una copia exacta de la imagen original en formato BMP.
  • JPEGHandler: Clase que representa la aplicacion principal, desde la cual se ejecutara el programa. Esta ya provee la opcion de generar una copia utilizando el BmpHandlerCopy, sin embargo debe ser modificada por su persona para agregar las opciones para archivos JPEG. Esta es solo para darle una guia de como deberia hacer con las demas.

Considerando lo anterior, su proyecto debe estar dividido en las siguientes clases:

  • Clase JPEGtoBMPImage, la cual se encargue de leer una imagen JPEG y genera una copia pero en formato BMP.
  • Clase BMPtoJPEGImage, la cual se encargue de leer una imagen BMP y genera una copia pero en formato JPEG.
  • Clase JPEGImageCopy, la cual se encargue de leer una imagen JPEG y crear una copia.
  • Clase JPEGImageHandlerColors, la cual se encargue de generar las imagenes especificadas en la parte III del proyecto.
  • Clase JPEGImageHandlerRotator, la cual se encargue de generar las imagenes especificadas en la parte IV del proyecto.
  • Clase principal llamada JPEGHandler, la cual unicamente se debe encargar de mandar a llamar a las clases anteriores para ejecutar las clases que se les especifiquen.

Las clases JPEGtoBMPImage, BMPtoJPEGImage, JPEGImageCopy, JPEGImageHandlerColors, y JPEGImageHandlerRotator deben heredar de la clase abstracta ImageHandler, e implementar los metodos abstractos de lectura y generacion de archivos, los cuales se mandan a ejecutar por el metodo runHandler de la clase JPEGHandler.

Forma de ejecutarlo

  • Como se habia dicho antes, la clase principal con la que se ejecutara su proyecto debe llamarse JPEGHandler (cuya primera version se le provee en el material del proyecto).
  • El programa debe ejecutarse como JPEGHandler seguido de, generalmente, una bandera y el nombre de la imagen original. La bandera indicara que parte del proyecto se va a ejecutar, es decir el modo de ejecucion.
  • Si la bandera es -tobmp su programa recibira un jpeg y devolvera un bmp (Parte I del proyecto). 
    		java JPEGHandler -tobmp imagen.jpeg
  • Si la bandera es -tojpeg su programa recibira un bmp y devolvera un jpeg (Parte I del proyecto). 
    		java JPEGHandler -tojpeg imagen.bmp
  • Si la bandera es -copy su programa solo debe ejecutar la Parte II del proyecto, es decir recibir un jpeg, realizar la copia y generar otro jpeg. 
    		java JPEGHandler -copy imagen.jpeg
  • Si la bandera es -colors su programa solo debe ejecutar la Parte III de este proyecto. 
    		java JPEGHandler -colors imagen.jpeg
  • Si la bandera es -rotate su programa debe ejecutar las rotaciones de la Parte IV de este proyecto. 
    		java JPEGHandler -rotate imagen.jpeg
  • Si la bandera es -all su programa debe ejecutar todas las partes de este proyecto, menos la parte II. 
    		java JPEGHandler -all imagen.jpeg
  • Si la bandera es -help, usted debe listar las opciones de como se debe ejecutar el JPEGHandler 
    		java JPEGHandler -help
Si usted no cumple con estos modos de ejecucion, tendra CERO en su proyecto.

En la clase JPEGHandler se le provee un ejemplo de como debe utilizar runHandler para mandar a llamar cada uno de los Handlers dependiendo de la opcion de ejecucion que se haya elegido.

Le recordamos que la version de JPEGHandler que se le provee, es una version basica que debe ser modificada por su persona (menos el metodo que no se permite modificar) para casar con las condiciones descritas con anterioridad. Por ningun motivo su programa debe terminar por alguna Exception, por lo que debe considerar los errores que puedan suceder, para atraparlos y hacer algo al respecto.

Al mandar a ejecutar el programa, se deben generar las imagenes y ser guardadas en el mismo directorio en que se ejecuto (No en directorios adicionales).

Observaciones:

Para realizar su proyecto debe tomar en cuenta lo siguiente y cumplir con las siguientes restricciones:

  • Trabajaremos SOLO con imagenes de 640X480.
  • Las unicas estructuras de datos que puede utilizar son: Arreglos, ArrayList y LinkedList.
  • Recuerdese que el formato va al reves, es decir que BMP empieza a contar desde el pixel inferior izquierdo.
  • Recuerde que en el formato BMP tiene dos bytes reservados (con valor cero) al final.
  • IMPORTANTE: NO PUEDE UTILIZAR CLASES EN JAVA (NI OTROS PROGRAMAS/LIBRERIAS YA HECHAS) QUE HAGAN ESTE TRABAJO POR USTED. NO PUEDE UTILIZAR CLASES EN JAVA QUE MANEJEN IMAGENES (lean atributos o modifiquen) a exepcion de la transformacion de JPEG a BMP y viceversa. Como por ejemplo la libreria: java.awt.Image (que incluye clases como BufferedImage), javax.imageIO, o la clase imageIO. Si hace esto, se le pondra cero en el proyecto. La manipulacion y generacion de la imagen debe estar programada por usted
  • Su proyecto NO debe implementar interfaz grafica, ni para puntos extra.
